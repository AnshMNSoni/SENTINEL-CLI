# 🎥 Sentinel Video Tutorial Series Plan

To maximize community reach and search-engine visibility, we'll produce three high-impact, 5-minute tutorials. These are designed to convert curious visitors into active users and contributors.

---

### Video 1: "The 60-Second Security Shield" (Install + First Scan)
**Goal**: Show that Sentinel is zero-friction and delivers instant value.

- **0:00 - 0:30**: Introduction — The problem (noisy security tools) vs. the solution (Sentinel's AI-powered local guardian).
- **0:30 - 1:30**: Installation via `npm install -g sentinel-cli`.
- **1:30 - 3:30**: Running `sentinel security-audit`. Show the interactive feedback and AI-powered fix suggestions.
- **3:30 - 4:30**: Briefly show the Web Dashboard — "Premium analytics for every developer."
- **4:30 - 5:00**: Outro & Call to Action (GitHub link).

---

### Video 2: "Unbreakable CI/CD" (GitHub Actions Integration)
**Goal**: Demonstrate how Sentinel prevents security regressions in production.

- **0:00 - 1:00**: Why CI/CD security matters.
- **1:00 - 3:00**: Live setup of `.github/workflows/sentinel.yml`. We'll show the 3-line integration.
- **3:00 - 4:00**: Triggering a Pull Request. Show the Sentinel Action posting inline review comments.
- **4:00 - 5:00**: Configuring blocking scans to ensure zero critical bugs reach `main`.

---

### Video 3: "Beyond Defaults" (Writing Custom Rules)
**Goal**: Empower developers to build their own guardrails.

- **0:00 - 1:00**: Introducing `.sentinelrules.yaml`.
- **1:00 - 3:00**: Live-coding a custom rule (e.g., "No unauthenticated API routes" or "Least-privilege S3 bucket patterns").
- **3:00 - 4:00**: Testing the rule with a dry-run scan.
- **4:00 - 4:30**: Shouting out the **Community Rules Cookbook** — how to share your rules with the world.
- **4:30 - 5:00**: Community & Discord invite.

---

### 🚀 Implementation Strategy
- **Recording Tools**: Use `asciinema` or `vhs` for crystal-clear terminal recordings.
- **Aesthetic**: Use a high-contrast terminal theme (e.g., Catppuccin or Dracula) with a modern font (e.g., JetBrains Mono).
- **Audio**: Keep it fast-paced, high energy, and technical.
- **Distribution**: Embed these in the `README.md` and the upcoming Mintlify site.
